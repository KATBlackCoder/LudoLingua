# Other Text Translation Guidelines

## For Note Fields:
1. **Tone Preservation**: Maintain the original tone and style of the text
2. **Formatting**: Preserve any special formatting, punctuation, or line breaks
3. **Formality Level**: Keep the same level of formality as the original
4. **Cultural Adaptation**: Ensure cultural appropriateness for the target audience
5. **Consistency**: Maintain consistency with similar text in the game
6. **Clarity**: Ensure the text remains clear and informative
7. **Context Preservation**: Keep important context and meaning intact

**Special Considerations for Note Fields:**
- Developer notes: Maintain technical information and references
- Hidden content: Preserve secret or bonus information
- Metadata: Keep important game data and references
- Comments: Maintain explanatory or contextual information

**Note Field Examples:**
- "Developer comment: This item is rare." => "Developer comment: This item is rare."
- "Hidden treasure location reference." => "Hidden treasure location reference."
- "Special event trigger condition." => "Special event trigger condition."

## For Metadata Fields:
1. **Technical Accuracy**: Maintain precise technical information
2. **Formatting**: Preserve any special formatting, codes, or structure
3. **Consistency**: Keep consistent with similar metadata in the game
4. **Clarity**: Ensure technical information remains clear
5. **Reference Preservation**: Keep important references and links intact

**Special Considerations for Metadata Fields:**
- Technical references: Maintain precise technical information
- Game data: Preserve important game mechanics and references
- System information: Keep system-related data accurate
- Reference links: Maintain important connections and references

**Metadata Field Examples:**
- "ID: 1001, Type: Weapon" => "ID: 1001, Type: Weapon"
- "Reference: Map_001, Event_005" => "Reference: Map_001, Event_005"
- "System: Battle_Trigger_01" => "System: Battle_Trigger_01"
